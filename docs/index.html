remote-dom

Server-authoritative DOM streaming for humans and agents

Encode the DOM as a structured op stream over WebSocket. Headless Chrome on the server, thin clients anywhere. The wire is semantic — not pixels, not video — so any consumer can parse, filter, record, or act on it.

Multiple clients connect to one session. Humans observe and interact. Agents read the DOM and dispatch actions. Same protocol, same state.

How it works

The server owns the browser. Clients are thin input/render terminals.

────────────────                 ──────                     ────────────────────

                            └─────────────────┘

Human B (phone/tablet)                                 Any WebSocket client
─────────────────────               │                  can connect:
                                    │                  browsers, scripts,
 Same session, synced ◄─MutationOp──┘                  agents, test runners

Key features

A framework for building on top of — not a finished app.

Core

DOM Streaming, Not Pixels

Streams structured DOM operations (childList, attributes, characterData) instead of pixel data. Orders of magnitude less bandwidth than VNC/RDP. Clients render natively.

Core

Headless Chrome Backend

Full browser engine via Puppeteer with stealth plugin. Real JavaScript execution, real layout, real network stack. Pages behave exactly as they would in a normal browser.

Architecture

Multi-Client Sessions

Multiple clients connect to the same browser session simultaneously. Everyone sees the same page state. Mix human observers with AI agent controllers.

Protocol

Clean JSON Protocol

Five op types: snapshot, childList, attributes, characterData, property. Easy to parse, easy to extend.

Core

Full Input Relay

Relay clicks, keyboard events, scroll, and navigation from any client. Clients can observe passively or actively control the page.

Architecture

Framework, Not an App

A building block. Write your own client renderers, agent loops, session managers, and automation pipelines on top of the protocol.

Get started

Up and running in minutes.

Install

terminal
npm install remote-dom

Start a session

Launch a headless browser, navigate to a page, and start streaming DOM ops.

server.js
import { RemoteDomServer } from 'remote-dom/server';

const server = new RemoteDomServer({ port: 3000 });

// Create a session — launches headless Chrome
const session = await server.createSession({
  url: 'https://example.com',
  headless: true,
});

console.log(`Session ${session.id} ready on ws://localhost:3000`);

Connect a client

Any WebSocket client can connect — a browser, a Node script, or an AI agent.

client.js
import { RemoteDomClient } from 'remote-dom/client';

const client = new RemoteDomClient('ws://localhost:3000');

// Receive DOM operations
client.on('snapshot', (html) => {
  document.getElementById('app').innerHTML = html;
});

client.on('mutation', (op) => {
  // op.type: 'childList' | 'attributes' | 'characterData' | 'property'
  applyMutation(op);
});

// Send input upstream
client.sendClick({ targetId: 'btn-submit', x: 100, y: 200 });
client.sendKeypress({ key: 'Enter', code: 'Enter' });

Build on top

Use the protocol to build agent loops, testing tools, collaborative browsing, or anything else.

agent.js
// AI agent example — observe DOM, decide, act
client.on('snapshot', async (html) => {
  const action = await llm.decide(html);

  if (action.type === 'click') {
    client.sendClick(action.target);
  } else if (action.type === 'type') {
    client.sendInput(action.target, action.text);
  }
});

The protocol

Simple JSON ops over WebSocket. Easy to implement in any language.

Server → Client

  • snapshot — full serialized DOM on connect
  • childList — nodes added or removed
  • attributes — attribute changed
  • characterData — text content changed
  • property — input value, checked, etc.

Client → Server

  • click / mousedown / mouseup
  • keydown / keyup / keypress
  • input — text field value
  • scroll — scroll position
  • navigate — go to URL
